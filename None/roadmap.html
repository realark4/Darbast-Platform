معماری Clean Architecture

  • API
    لایه‌ ارائه (Routers) مدیریت Endpointهای FastAPI، دریافت ریکوئست‌ها و اعتبارسنجی با Pydantic.
  • Core
    هسته و تنظیمات (Settings/Security) مدیریت JWT، هش کردن پسوردها، تنظیمات Pydantic Settings و CORS.
  • Services
    منطق تجاری (Business Logic) اتصال به Qdrant، ساخت Embeddingها، و قوانین تجاری (مثل بررسی مالکیت آگهی).
  • DB
    لایه دسترسی داده (Models/CRUD) مدل‌های SQLAlchemy و توابع عملیاتی پایگاه داده اصلی (PostgreSQL/SQLite).

اسکیمای دیتابیس (ERD)

Table users {
  id int [pk]
  username varchar
  email varchar [unique]
  hashed_password varchar
  is_active boolean
}

Table properties {
  id int [pk]
  owner_id int [ref: > users.id]
  title varchar
  description text
  price decimal
  city varchar
  type varchar // اجاره، فروش
  created_at timestamp
}
نکته تخصصی (SQLAlchemy)

برای جلوگیری از مشکل N+1 در خواندن لیست آگهی‌ها به همراه اطلاعات مالک، از joinedload در کوئری‌های خود استفاده کنید.

معماری Semantic Search & RAG

1

تولید Embedding

هر بار که آگهی جدیدی ثبت یا ویرایش می‌شود، اطلاعات آن توسط یک مدل هوش مصنوعی (مثل sentence-transformers) به یک وکتور عددی تبدیل می‌گردد.

2

ذخیره در Qdrant

وکتور تولید شده همراه با متادیتا (مثل قیمت و شهر) در دیتابیس Qdrant ذخیره می‌شود که امکان جستجوی سریع شباهت معنایی را فراهم می‌کند.

3

چت‌بات RAG

سوال کاربر Embed شده، آگهی‌های مرتبط از Qdrant پیدا شده و به عنوان Context به LLM داده می‌شود تا پاسخ دقیق و هوشمندانه‌ای برای کاربر تولید کند.

مثال جستجوی ترکیبی (Hybrid Search):

qdrant_client.search(
    collection_name="properties",
    query_vector=question_embedding,
    query_filter=models.Filter(
        must=[models.FieldCondition(key="city", match=models.MatchValue(value="Tehran"))]
    ),
    limit=5
)
